Task 3: Test Case Creation

This document outlines test cases for the Before & After Image Manager WordPress plugin, covering functional, performance, and edge case scenarios.

Test Case 1 – Functional Test: Image Upload Validation

Objective: Verify that the plugin correctly validates image dimensions (minimum 500x500 px).

Preconditions:

  • User has access to the WordPress admin panel.
  • Plugin is installed and activated.

Steps:

  1. Go to Before & After Gallery → Add New Case.
  2. Try uploading an image smaller than 500x500 px (e.g., 400x400).
  3. Observe the error or validation message.
  4. Upload an image with correct dimensions (e.g., 800x800).
  5. Save the case.

Expected Results:

  • Step 2: Upload fails with message “Image must be at least 500x500 pixels.”
  • Step 5: Upload succeeds, image is saved and displayed correctly.

Type: Functional
Priority: High
Status: Pass/Fail

Test Case 2 – Performance Test: Gallery Load Speed

Objective: Ensure that gallery pages load efficiently when displaying multiple before/after cases.

Preconditions:

  • Plugin is configured with at least 20 before/after cases.
  • Caching enabled (if available).

Steps:

  1. Open the gallery page on the front end.
  2. Measure page load time using Lighthouse or GTmetrix.
  3. Compare load time against acceptable threshold (≤3 seconds).

Expected Results:

  • Page loads within 3 seconds on standard broadband.
  • No console or network errors.
  • Smooth image transitions or sliders with no lag.

Type: Performance
Priority: Medium
Status: Pass/Fail

Test Case 3 – Edge Case: Category Ordering Logic

Objective: Validate that cases display correctly when assigned to both primary and secondary categories.

Preconditions:

  • At least 3 cases exist.
  • Each case has a primary and secondary category.

Steps:

  1. Assign Case A as primary category “Skincare”, secondary “Face”.
  2. Assign Case B as primary “Skincare”, secondary “Body”.
  3. Reorder cases under “Skincare”.
  4. Open the “Face” and “Body” galleries.

Expected Results:

  • Case order in secondary categories reflects the same order set in the primary category.
  • No duplication or missing cases.

Type: Edge Case / Functional
Priority: High
Status: Pass/Fail
