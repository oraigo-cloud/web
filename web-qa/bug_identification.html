Task 2: Bug Identification and Reporting

This document outlines how to identify, describe, and report bugs found during QA testing of a WordPress website.

1. Contact Form Not Submitting Data

Root Cause Investigation

  • Check browser console for JavaScript or AJAX errors during submission.
  • Use the Network tab to inspect POST requests and server responses.
  • Review wp-content/debug.log for PHP or plugin errors.
  • Test for plugin conflicts by disabling others or switching to a default theme.
  • Verify form settings: endpoint, email setup, and reCAPTCHA configuration.

Description for Developer

When submitting the contact form, no confirmation message appears and data is not received. The POST request fails, likely due to an AJAX or backend validation error.

2. Homepage Slider Breaks on iPhone 13 (Safari)

Root Cause Investigation

  • Reproduce on iPhone 13 Safari using BrowserStack or a real device.
  • Check console errors via remote debugging.
  • Inspect CSS and JS for compatibility issues (e.g., object-fit, flex, IntersectionObserver).
  • Verify slider plugin version and responsive settings.
  • Test disabling custom JS to isolate conflicts.

Description for Developer

The homepage slider fails to display correctly on iPhone 13 Safari. Slides overlap or disappear, likely due to a mobile-specific JS or CSS bug.

3. Page Load Speed Exceeds 6 Seconds

Root Cause Investigation

  • Run Lighthouse/PageSpeed Insights for bottlenecks (images, render-blocking scripts).
  • Check hosting response time (TTFB) using WebPageTest.
  • Use Query Monitor for slow queries or excessive requests.
  • Review caching, CDN, and image compression setup.

Description for Developer

Homepage takes over 6 seconds to load. Performance audit shows large unoptimized images and multiple blocking JS files. Needs optimization for caching, images, and JS delivery.

4. Header Navigation Fails WCAG Contrast Requirements

Root Cause Investigation

  • Use Axe or WAVE to confirm contrast ratio.
  • Check CSS variables or theme colors applied to links.
  • Test normal and hover states for all navigation links.
  • Review style.css or SCSS color assignments.

Description for Developer

Header navigation links have insufficient contrast (2.8:1) against background. WCAG 2.1 AA requires at least 4.5:1. Adjust link colors to meet contrast standards.

Sample Bug Report (for Developer)

Bug ID: QA-004
Title: Contact form not submitting data on staging site

Environment:
URL: https://staging.example.com/contact
Browser: Chrome 129, macOS
WP Version: 6.5
Plugin: Contact Form 7 (customized)

Description:
When submitting the contact form, the page reloads without showing a success message, and no data/email is received.

Steps to Reproduce:
1. Go to /contact
2. Fill in all fields with valid data
3. Click “Submit”

Expected Result:
Form submits successfully, shows a “Thank you” message, and sends email notification.

Actual Result:
Form reloads; no confirmation message or email received.

Console / Network Evidence:
Console shows Uncaught TypeError: Cannot read property 'response' of undefined.
Network tab: POST request returns 500 error.

Severity: High (Form submission broken)
Priority: P0 — Must fix before release

Suggested Fix:
Investigate custom AJAX handler or plugin conflict; verify form endpoint and nonce validation.

Attachments:
Screenshot: contact-form-error.png
Console log: form_submit_error.txt
