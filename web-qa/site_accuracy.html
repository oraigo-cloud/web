Design Comp vs. Dev Site Accuracy

This section outlines the approach to verify that the client’s design comp has been accurately implemented in the WordPress dev site.

1. Approach to Verifying Design Accuracy

Step 1: Prepare

  • Obtain final approved design comps (Figma, Adobe XD, Sketch).
  • Ensure access to the staging/dev site.
  • Note design specs: font families, sizes, colors (HEX/RGB), padding, margins, button states, etc.

Step 2: Visual Comparison

  • Compare design comp side-by-side with dev site pages.
  • Verify the following:
    • Typography: Font family, weight, size, line-height, color.
    • Layout: Section spacing, alignment, grid structure.
    • Colors: Backgrounds, text, buttons, hover states.
    • Interactive Elements: Buttons, sliders, links, hover/focus behavior.
    • Images & Icons: Correct assets, positioning, proportions.
  • Check both desktop and mobile views for responsive accuracy.

Step 3: Validation & Documentation

  • Capture screenshots showing design vs. dev differences.
  • Use measurement tools to compare spacing and size.
  • Log discrepancies in a QA tracker (Jira, Excel, Trello).
  • Classify issues by severity:
    • High: Breaks layout or brand guidelines.
    • Medium: Minor spacing or color mismatch.
    • Low: Subtle alignment or typographic variation.

✅ Summary:

  • Use design specs and pixel comparison tools for accuracy.
  • Test across breakpoints for responsive fidelity.
  • Log and prioritize deviations based on visual or functional impact.
  • Re-test after fixes to confirm alignment with design.
